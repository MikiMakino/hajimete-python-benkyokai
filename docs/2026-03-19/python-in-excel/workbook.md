# Lesson 1：はじめて `=PY()` を使う

## このレッスンでやること

まず Python in Excel が動くことを確認します。
まだ売上表は使いません。最初は「動かせる」と体験するのが目的です。

---

## 1-1. 数字を計算してみる

### やってみる

空いているセルに、次の式を入力してください。

```excel
=PY("1+2")
```

### 結果の見方

結果として **3** が表示されれば成功です。

### ポイント

この式では、Excel の中で Python の式 `1+2` を実行しています。
そして、その結果が Excel に返されています。

最初は難しく考えなくて大丈夫です。
大切なのは、**「Excel の中で Python が動かせた」と確認できること** です。

### かえてみる

次の式も試してみましょう。

```excel
=PY("10*5")
=PY("100/4")
```

---

## 1-2. 文字を返してみる

### やってみる

次の式を入力してください。

```excel
=PY("'Hello Python in Excel'")
```

### 結果の見方

文字がそのまま表示されれば成功です。

### ポイント

Python in Excel では、数字だけでなく文字も返します。
つまり、Python の処理結果をそのまま Excel に表示できるということです。

### かえてみる

好きな文字に変えて試してみましょう。

```excel
=PY("'売上データ分析スタート'")
```

---

## 1-3. このレッスンのまとめ

このレッスンで大切なのは、次の1点です。

> `=PY()` を使うと、Excel の中で Python が動かせる

まだ文法を覚える必要はありません。
まずは「動かせた」という感覚をつかめれば十分です。

# Lesson 3：基本集計してみる

## このレッスンでやること

このレッスンでは、売上表から必要な数字を取り出します。
ここで学ぶのは、**「列を指定して計算する」** という感覚です。

---

## 3-1. 売上合計を出してみる

**問い：** 今月の売上合計は、いくらでしょうか？

### やってみる

```python
sales_tbl["売上"].sum()
```

### 結果の見方

```
35700
```

### ポイント

この式では、売上表の中の **「売上」列全体** に対して合計を求めています。
Excel でいうと SUM に近いですが、Python では **列に対して命令する** 感じになります。

### かえてみる

数量の合計を計算してみましょう。

```python
sales_tbl["数量"].sum()
```

---

## 3-2. 平均を出してみる

**問い：** 1件あたりの平均売上はいくらでしょうか？

### やってみる

```python
sales_tbl["売上"].mean()
```

### 結果の見方

平均値が表示されれば成功です。
結果が小数になることがあります。

### ポイント

ここでも「売上」列全体が対象です。
1件ずつ計算しているわけではなく、**列全体から平均を求めている** のがポイントです。

### かえてみる

単価の平均を出してみましょう。

```python
sales_tbl["単価"].mean()
```

---

## 3-3. 最大値と最小値を見てみる

**問い：** 最も大きい売上は？　最も小さい売上は？

### やってみる

最大値：

```python
sales_tbl["売上"].max()
```

最小値：

```python
sales_tbl["売上"].min()
```

### ポイント

集計といえば合計や平均が浮かびがちですが、最大値・最小値もデータの傾向を見るのにとても役立ちます。

たとえば、

- 一番高い売上はどれか
- 一番小さい売上はどれか

すぐに把握できます。

### かえてみる

数量列について、最大値と最小値を出してみましょう。

```python
sales_tbl["数量"].max()
sales_tbl["数量"].min()
```

---

## 3-4. 件数を数えてみる

**問い：** 売上データは全部で何件ありますか？

### やってみる

```python
sales_tbl["売上"].count()
```

### 結果の見方

件数が数字で表示されれば成功です。

### ポイント

この表には何行あるか、つまりデータが何件あるかを確認できます。
件数は、後で条件で絞り込んだデータを数えるときにも役立ちます。

---

## 3-5. ここまでの整理

このレッスンで使った操作をまとめます。

| 操作 | 意味 |
|------|------|
| `sum()` | 合計 |
| `mean()` | 平均 |
| `max()` | 最大値 |
| `min()` | 最小値 |
| `count()` | 件数 |

大切なのは、関数名を全部暗記することではありません。
**「列に対して、こういう計算ができる」** とわかることが重要です。

---

## 3-6. このレッスンのまとめ

- 売上表の中の列を指定して集計できた
- 合計・平均・最大値・最小値・件数が求められた
- Python in Excel は、表の集計の入口として使いやすい

# Lesson 4：条件で絞り込む

## このレッスンでやること

このレッスンでは、条件に合うデータだけを取り出します。
Excel のフィルターに似ていますが、Python では **条件を式として書く** のが特徴です。

---

## 4-1. 東京支店だけを取り出してみる

**問い：** 東京支店の売上だけを見たいとき、どうすればいいでしょうか？

### やってみる

```python
sales_tbl[sales_tbl["支店"] == "東京"]
```

### 結果の見方

東京支店の行だけが表示されれば成功です。

### ポイント

この式は、**「支店列が "東京" である行だけを取り出す」** という意味です。

大切な考え方は：

1. 表全体の中から
2. 条件に合う行だけを選ぶ

### かえてみる

大阪支店だけを取り出してみましょう。

```python
sales_tbl[sales_tbl["支店"] == "大阪"]
```

---

## 4-2. A商品だけを取り出してみる

**問い：** A商品のデータだけを見たいときは、どうすればいいでしょうか？

### やってみる

```python
sales_tbl[sales_tbl["商品"] == "A商品"]
```

### ポイント

今回は支店ではなく商品列を条件にしています。
つまり、**どの列を条件にするかを変えるだけ** で、見たいデータを取り替えられます。

### かえてみる

B商品だけを取り出してみましょう。

```python
sales_tbl[sales_tbl["商品"] == "B商品"]
```

---

## 4-3. 売上が大きい行だけを取り出してみる

**問い：** 売上が3000円以上のデータだけを見たいときは？

### やってみる

```python
sales_tbl[sales_tbl["売上"] >= 3000]
```

### ポイント

今度は文字ではなく、**数値条件** で絞り込んでいます。
Python では、文字条件と数値条件の両方が使えます。

### かえてみる

```python
sales_tbl[sales_tbl["売上"] >= 5000]
```

---

## 4-4. 条件に合う件数を数えてみる

**問い：** 東京支店のデータは何件ありますか？

### やってみる

```python
sales_tbl[sales_tbl["支店"] == "東京"]["売上"].count()
```

### ポイント

これは次の2段階で考えるとわかりやすいです。

1. 東京支店の行だけを取り出す
2. そのデータの件数を数える

つまり、**絞り込みと集計を組み合わせている** ということです。

---

## 4-5. 2つの条件を組み合わせてみる

**問い：** 東京支店で、売上が3000円以上のデータだけを見たいときは？

### やってみる

```python
sales_tbl[(sales_tbl["支店"] == "東京") & (sales_tbl["売上"] >= 3000)]
```

### ポイント

ここでは条件を2つ組み合わせています。
最初は少し複雑に見えるかもしれませんが、考え方は同じです。

- 条件1：支店が東京
- 条件2：売上が3000以上
- 両方に当てはまる行を取り出す

このように、条件を増やして表を見ながら考えられるのが Python の便利なところです。

---

## 4-6. このレッスンのまとめ

- 条件に合う行だけを取り出せた
- 文字条件と数値条件の両方が使えた
- 絞り込んだあとに件数を数えられた
- 条件を組み合わせることもできた

# Lesson 7：つまずきやすいこと

## 1. 列名が合わない

たとえば、次のようなエラーが出ることがあります。

- 列名の入力ミス
- 全角と半角のちがい
- 見出しの文字が少しちがう

**確認ポイント**

- 見出しは本当に `"売上"` か
- `"支店"` `"商品"` などの文字が一致しているか
- 余計な空白が入っていないか

---

## 2. テーブル名がちがう

このワークブックでは `sales_tbl` を使っています。
テーブル名が違うと、そのままでは動かないことがあります。

**確認ポイント**

- テーブル名が `sales_tbl` になっているか
- 別の名前にしている場合は、その名前に読み替えて使う

---

## 3. 記号の向きや数がちがう

たとえば、次のようなちがいに注意します。

| まちがいやすい例 | 正しい例 |
|---|---|
| `=` | `==` （条件判定は `==`） |
| `=>` | `>=` （以上は `>=`） |
| `'` と `"` の混在 | 統一して使う |

---

## 4. カッコが足りない

条件を複数組み合わせるときは、カッコが足りないとエラーになることがあります。

```python
# 正しい例
sales_tbl[(sales_tbl["支店"] == "東京") & (sales_tbl["売上"] >= 3000)]
```

最初は丸ごとコピーして試して大丈夫です。

---

## 5. 結果が思った形とちがう

こういうときは、次の順番で確認すると整理しやすくなります。

1. どの列を見ているか
2. どんな条件をつけているか
3. 最後に何をしたいのか（表示 / 件数 / 合計 など）

# 付録A：よく使う操作一覧

## 基本の集計

```python
# 売上合計
sales_tbl["売上"].sum()

# 平均
sales_tbl["売上"].mean()

# 最大値
sales_tbl["売上"].max()

# 最小値
sales_tbl["売上"].min()

# 件数
sales_tbl["売上"].count()
```

## 条件で絞り込む

```python
# 東京支店だけ
sales_tbl[sales_tbl["支店"] == "東京"]

# A商品だけ
sales_tbl[sales_tbl["商品"] == "A商品"]

# 売上3000以上
sales_tbl[sales_tbl["売上"] >= 3000]

# 東京支店の件数
sales_tbl[sales_tbl["支店"] == "東京"]["売上"].count()

# A商品の平均売上
sales_tbl[sales_tbl["商品"] == "A商品"]["売上"].mean()
```

## 2つの条件を組み合わせる

```python
# 東京支店 かつ 売上3000以上
sales_tbl[(sales_tbl["支店"] == "東京") & (sales_tbl["売上"] >= 3000)]
```
